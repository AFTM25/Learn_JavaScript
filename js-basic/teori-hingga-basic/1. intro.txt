--- Apa itu pemrograman ---
1. Program adalah serangkaian instruksi pada komputer. 
2. Apa itu instruksi untuk komputer? 
  > Memberi perintah pada komputer 
  > Melalui bahasa pemrograman 
  > Memecah tugas jadi instruksi sederhana. Seperti apa instruksi sederhana itu?
    - Melakukan operasi matematika 
    - Menampilkan tulisan ke layar
    - Mengubah ukuran, posisi atau warna pada sebuah pixel.

3. Perilaku komputer : 
  - Komputer tidak begitu pintar
  - Tapi dia penurut (akan melakukan instruksi apa pun yang kita berikan meski salah)
  - Komputer tidak kenal lelah 
  "Berikanlah instruksi yang benar"
  Instruksi yang kita berikan pada komputer disebut statement. 

Ringkasan : 
Apa itu pemrograman?
- Cara kita memberikan instruksi yang simpel dan terstruktur pada komputer 
- Instruksi yang diberikan haruslah benar dan masuk akal 
- Instruksi harus sesuai dengan aturan pada bahasa pemrograman tertentu.  


--- Apa itu bahasa pemrograman? ---
Kenapa banyak bahasa pemrograman? 
- Awal komputer menggunakan bahasa mesin yakni binary. 
- Sulit dipahami manusia 
- Dibuatlah bahasa pemrograman tingkat tinggi yang lebih mudah dipahami manusia.

Pembahasan mengenai source code : 
1. Bagaimana menulis source code? 
  - Menggunakan plain text atau teks editor. 
  - Hal ini berbeda dengan software pengolah kata (word processor) seperti microsoft word.
  Masalah : Apakah kita harus menghapalkan blok kode program atau sintaks? 
  : Jika ya, ketika menggunakan notepad 
  : Tidak, karena ada software yang bisa membantu untuk menuliskan sintaks seperti kode editor. 

2. Bagaimana source code diterjemahkan jadi machine code? 
  - Compiler    : 
    - Merupakan  program yang menterjemahkan dari source code ke machine code sekaligus, sebelum dijalankan. 
    - Proses (melalui 6 tahap) : 
      1. Lexical Analysis = memecah kode jadi token(variabel, keyword, simbol)
      2. Syntax Analysis = memastikan struktur kode valid (pakai aturan grammar bahasa itu)
      3. Semantic Analysis = memastikan makna nya logis (misal:ga bisa jumlahin string sama integer) 
      4. Optimization = mempercepat eksekusi tanpa merubah hasil
      5. Code Generation = ubah jadi assembly/machine code
      6. Code Linking = gabung semua file yang dibutuhkan jadi file execurtable(.exe, .out, dll)
    - Gambaran proses compiler : 
      script.c  --> Compiler -->  program.exe --> run

  - Interpreter : 
    - Merupakan program yang menterjemahkan source code baris per baris saat program dijalankan, tanpa mengubah dulu ke file executable.
    -Proses : 
      - Baca 1 baris 
      - Ubah ke instruksi mesin 
      - Jalankan
      - Lanjut ke baris berikutnya sampai selesai 
    - Gambaran proses interpreter :
      script.py --> Interpreter --> run

  - Perbedaan compiler dan interpreter
    - Compiler : 
      - Kelebihan : 
        - Siap untuk dijalankan 
        - Lebih cepat 
        - Sorce code tidak terlihat 
      - Kekurangan :
        - Tidak lintas platform
        - tidak fleksibel
        - langkah tambahan 
    
    -Interpreter : 
      - Kelebihan : 
        - Lintas platform
        - Fleksibel
        - Mudah untuk diuji 
      
      - Kekurangan :
        - Perlu interpreter terinstall
        - Lebih lambat
        - Source code terlihat dan mudah diakses

  - Hybrid (gabungan compiler interpreter)
    - Intermediete Language (IL)
    - JIT compilation (Just-In-Time)
      - Menggabungkan kelebihan compiler dan interpreter
      - Melakukan kompilasi saat program dijalankan
      - Meningkatkan performa eksekusi program
